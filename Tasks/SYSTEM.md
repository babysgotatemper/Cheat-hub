# 📋 Task Management System

## Як це працює

### 1️⃣ **Ви даєте мені задачу**
```
Ви: "Додай всі задачі з LeetCode"
```

### 2️⃣ **Я автоматично створю MD-файл із планом**
```
Tasks/task-NNN-task-name.md ← з повним планом виконання
```

### 3️⃣ **Я виконую задачу крок за кроком**
- Читаю план з MD-файлу
- Виконую кожен крок
- Перевіряю результати

### 4️⃣ **Результати записуються в .done.md файл**
```
Tasks/task-NNN-task-name.done.md ← з результатами виконання
```

---

## 📂 Структура файлів

```
Tasks/
├── README.md                           # Інструкція (цей файл)
├── SYSTEM.md                           # Опис системи
├── TEMPLATE.md                         # Шаблон для нових задач
├── .task-counter                       # Лічильник для ID задач
│
├── task-001-import-leetcode-problems.md      # Опис задачі
├── task-001-import-leetcode-problems.done.md # Результати (створюється після)
│
├── task-002-add-filters.md
├── task-002-add-filters.done.md
│
└── task-NNN-*.md
    └── task-NNN-*.done.md
```

---

## 📝 Формат MD-файлу задачі

```markdown
# Task [ID]: [Назва]

**Дата:** YYYY-MM-DD
**Статус:** 📋 TODO / 🔄 IN_PROGRESS / ✅ DONE

## Опис
Коротко що робити

## План виконання

### Крок 1: [Назва]
- [ ] Підетап 1
- [ ] Підетап 2

### Крок 2: [Назва]
- [ ] Підетап 1

## Файли для зміни
| Файл | Дія |
|---|---|
| src/file.ts | НОВИЙ / ПЕРЕРОБИТИ / ВИДАЛИТИ |

## Результати
[Заповнюється після виконання]

## Примітки
Обмеження, залежності, тощо
```

```markdown

- Отримано cookie
- Додано в .env

- Динамічний dispatch працює

...

## 📊 Статистика

- Завантажено: 2547
- Помилок: 3
- Час: 18 хвилин 32 секунди

## 📸 Скріншоти/Доказ
[Результати перевірок та тестів]
```

## 💾 Зберігання історії

Всі задачи та результати зберігаються:
```
Tasks/
└── task-NNN-name.md       ← Оригінальний план
└── task-NNN-name.done.md  ← Результати
```

**История не видаляється** — можна переглянути що робилось раніше.

## 🔢 Автоматичний лічильник ID

Кожна нова задача отримує унікальний ID:
```
task-001-...
task-002-...
task-003-...
```

ID зберігається в `.task-counter` файлі.

## 🔧 Технічна реалізація

Система працює так:

1. **Створення:**
   - Я читаю твою задачу
   - Генерую MD-файл з планом
   - Збираю в папку `Tasks/`

2. **Виконання:**
   - Читаю план з MD-файлу
   - Виконую кроки за кроками
   - Логую прогрес

3. **Результати:**
   - Записую результати в `.done.md`
   - Фіксую статус ✅/❌
   - Зберігаю доказ (скріншоти, логи)
